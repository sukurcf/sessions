Discover & Install Plugins

Multiple ways to find and add plugins to your workspace

🔍
Discovery Methods

  • @agentPlugins — Search in Extensions sidebar (⇧⌘X)
  • Install from Source — Command Palette → Chat: Install Plugin From Source
  • Local Plugins — Register via chat.pluginLocations setting
  • Enable/Disable — Context menu in Installed panel

🏪
Plugin Marketplaces

Default: github/copilot-plugins and github/awesome-copilot

settings.json "chat.plugins.marketplaces": [ "anthropics/claude-code", "your-org/plugin-marketplace" ]

Formats: owner/repo, HTTPS .git URL, SCP-style, file:/// URI. Private repos supported.

PostToolUse After tool completes
SubagentStart Subagent is spawned

Hook Config & MCP Servers

Configuration examples for lifecycle hooks and external tools

⚙️
hooks.json

hooks/hooks.json { "hooks": { "SessionStart": [{ "type": "command", "command": "${CLAUDE_PLUGIN_ROOT}/scripts/session-log.sh", "timeout": 5 }], "PostToolUse": [{ "matcher": "Write|Edit", "hooks": [{ "type": "command", "command": "npx prettier --write \"$TOOL_INPUT_FILE_PATH\"" }] }] } }

🔌
MCP Servers

Bundle MCP servers to provide agents with additional tools. Auto-start on enable.

.mcp.json { "mcpServers": { "plugin-database": { "command": "${CLAUDE_PLUGIN_ROOT}/servers/db-server", "args": ["--config", "${CLAUDE_PLUGIN_ROOT}/config.json"] }, "plugin-api": { "command": "npx", "args": ["@company/mcp-server"], "cwd": "${CLAUDE_PLUGIN_ROOT}" } } }

Top-level key: mcpServers (not servers)

What Are Agent Plugins?

Preview

Pre-packaged bundles of chat customizations you can discover and install from plugin marketplaces in VS Code. One install → everything you need.

📦 Install 1 Plugin → 🤖 Agents ⚡ Skills 🔌 MCP 🪝 Hooks ⌨️ Commands
📌 Enable: Set chat.plugins.enabled: true in VS Code settings.
🎁

What Plugins Provide

TypeDescription
Slash CommandsAdditional commands you invoke with / in chat
SkillsAgent skills with instructions, scripts & resources — load on demand
AgentsCustom agents with specialized personas & tool configurations
HooksShell commands executing at agent lifecycle points
MCP ServersExternal tool integrations via Model Context Protocol
🔍

Discover & Install Plugins

@agentPlugins Search in Extensions sidebar (⇧⌘X)
Install from Source Command Palette → Chat: Install Plugin From Source with Git URL
Local Plugins Register via chat.pluginLocations setting with path → enabled/disabled
Enable/Disable Context menu in Agent Plugins - Installed or Chat Customizations editor
⚠️ Caution: Plugins can include hooks and MCP servers that run code on your machine. Review the plugin contents and publisher before installing.
🏪

Plugin Marketplaces

Default: github/copilot-plugins and github/awesome-copilot. Add more via settings.

settings.json
"chat.plugins.marketplaces": [
    "anthropics/claude-code",
    "your-org/plugin-marketplace"
]

Supported formats: owner/repo, HTTPS .git URL, SCP-style, file:/// URI. Private repos supported.

4 · Hooks & MCP Servers

🪝

Hooks — Lifecycle Automation

Preview

Execute shell commands at key agent lifecycle points for deterministic enforcement.

EventUse Case
SessionStartInject project context, audit logging
UserPromptSubmitValidate or transform user input
PreToolUseBlock dangerous operations
PostToolUseRun formatters, linters after edits
SubagentStart/StopMonitor subagent lifecycle
StopGenerate reports, cleanup
⚙️

Hook Configuration

hooks/hooks.json
{
  "hooks": {
    "SessionStart": [{
      "type": "command",
      "command": "${CLAUDE_PLUGIN_ROOT}/scripts/session-log.sh",
      "timeout": 5
    }],
    "PostToolUse": [{
      "matcher": "Write|Edit",
      "hooks": [{
        "type": "command",
        "command": "npx prettier --write \"$TOOL_INPUT_FILE_PATH\""
      }]
    }]
  }
}
📌 Note: Use ${CLAUDE_PLUGIN_ROOT} to reference scripts within the plugin. VS Code expands it at runtime.
🔗

Hook Interactions

  • Plugin hooks run alongside workspace & user-level hooks
  • Multiple hooks on same event → all execute
  • PreToolUse permissions: most restrictive wins (deny > ask > allow)
  • Disabling a plugin also disables its hooks
💡 Tip: Use PreToolUse hooks to block dangerous shell commands like rm -rf /, DROP TABLE.
🔌

MCP Servers in Plugins

Bundle MCP servers to provide agents with additional tools and data sources. Auto-start on plugin enable, stop on disable.

.mcp.json
{
  "mcpServers": {
    "plugin-database": {
      "command": "${CLAUDE_PLUGIN_ROOT}/servers/db-server",
      "args": ["--config", "${CLAUDE_PLUGIN_ROOT}/config.json"]
    },
    "plugin-api": {
      "command": "npx",
      "args": ["@company/mcp-server"],
      "cwd": "${CLAUDE_PLUGIN_ROOT}"
    }
  }
}
📌 Key: Top-level key is mcpServers (not servers). Plugin MCP servers are implicitly trusted — no separate trust prompt.
