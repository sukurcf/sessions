Understanding Code Scanning Alerts

Critical / High

Remote code execution, SQL injection, XSS — must fix before merge. Blocks PR if configured.

Medium

SSRF, path traversal, insecure deserialization — fix in current sprint. Should not accumulate.

Low / Note

Informational findings, code quality hints — review and address when convenient.

Alert Anatomy

  • Vulnerability type & CWE reference
  • Exact file, line number, and data flow path
  • Explanation of why the code is vulnerable
  • Suggested remediation guidance

Where Alerts Appear

  • Pull request checks & conversation tab
  • Repository Security tab
  • Security Overview dashboard (org-wide)
  • Email & webhook notifications

Copilot Autofix — How It Works

CodeQL Alert
→
SARIF + Code Context
→
LLM Analysis
→
Fix Suggestion

What Autofix Sends to LLM

  • CodeQL alert data (SARIF format)
  • Surrounding code snippets for context
  • Query help text explaining the vulnerability
  • Internally tested before display

Developer Review Steps

  • Review the suggested code change
  • Edit the fix if needed
  • Commit the change to the PR
  • Verify CI passes, then merge

Autofix — Quality & Limitations

✅ Strengths

  • Majority of alerts receive a fix suggestion
  • Most fixes can be committed as-is or with minor edits
  • Each suggestion is internally tested before display
  • No Copilot subscription required
  • Data is never used for LLM training
  • Red teaming + filters prevent harmful suggestions

⚠️ Limitations

  • Non-deterministic (LLM-based)
  • Complex multi-file flows may be difficult
  • Large files may have truncated context
  • May suggest partial or syntactically incorrect fixes
  • Could suggest fabricated/insecure dependencies
  • Always validate before merging
⚠️ Important: Copilot Autofix is an assistant, not a replacement for developer judgment. Always review, test, and validate suggested fixes before merging.

Secret Protection & Push Protection

What happens when you accidentally push a secret

git push
→
🛑 Push Blocked
→
Review & Decide
→
Remove Secret

Remove It

Best option — remove the secret from your commit, rotate the credential, and push clean code.

Bypass

Provide a reason: "used in tests", "false positive", or "fix later". Creates an alert for tracking.

Request Review

If delegated bypass is enabled, your bypass request goes to designated reviewers for approval.

💡 Pro Tip: User push protection is enabled by default for all GitHub.com users on public repos — even without a GHAS license.

Supply Chain Security

Dependency Graph

Automatically maps all direct and transitive dependencies across your repository. Supports npm, pip, Maven, NuGet, Go, Cargo, and more.

Dependabot Alerts

Notifies you when a dependency has a known vulnerability (CVE). Automatically opens PRs to update to a patched version.

Dependency Review

Shows dependency changes in every PR — new vulnerabilities introduced, license changes, and ecosystem impact before merge.

Auto-Triage Rules

Custom rules to automatically dismiss low-risk Dependabot alerts based on ecosystem, severity, scope, or package name patterns.
