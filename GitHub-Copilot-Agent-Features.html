Trust Boundaries & Security

🏗️ Trust Boundaries

WorkspaceUntrusted → restricted mode, agents disabled
ExtensionTrust publisher before activation
MCP ServerTrust prompt before first run
NetworkTrust domain before fetching content

🔒 Security Best Practices

  • Use dev containers for isolated environments
  • Review tool calls before approving actions
  • Scope file access to workspace boundaries
  • Leverage session-scoped permissions
  • Use request limits for cost control

🏢 Enterprise

Centralized policies: disable agents, restrict MCP, require manual approval

📦 Scope Isolation

Workspace-limited file access, session-scoped permissions, request limits

🔑 Secrets

Secure credentials store for MCP auth tokens & sensitive parameters

💡 Best Practice: Use a dev container for maximum security isolation. Review auto-approval rules carefully — shell aliases and complex syntax may bypass them.

Custom Instructions GA

Guide agent behavior with persistent rules — always-on context for every conversation

🏢 Organization

GitHub org-level instructions via admin settings. Applied to all repos & users in the org.

📁 Repository

.github/copilot-instructions.md
Shared with all contributors via version control.

👤 User-Level

VS Code settings or ~/.copilot/ instructions. Personal preferences across all projects.

📝 Prompt Files (.prompt.md)

  • Reusable prompt templates in .github/prompts/
  • Run as slash commands: /my-prompt
  • Support variables & file references
  • mode: agent|ask|edit front-matter

🎯 applyTo Patterns

  • Scope instructions to specific files or languages
  • Use glob patterns: **/*.py, src/**
  • Only loads when matching files are in context
  • Supports front-matter in .instructions.md

Common Troubleshooting

🔍 AI Ignores Files

Check Discovery events → verify workspace indexing → add #file or #codebase mentions

🔧 MCP Tool Not Called

Check Tool calls filter → verify tool in System prompt → reference with #tool-name

✂️ Response Cut Off

Check LLM request token usage → full context window triggers truncation → start new session

📝 Prompt File Not Applied

Check Discovery events for load/skip/fail → verify applyTo pattern matches context
