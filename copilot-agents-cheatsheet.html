Custom Agents GA

Specialized AI personas with defined behavior, tools, and model preferences.

📄 .agent.md — define persona, tools, model
↓
🧠 Copilot loads agent profile
↓
💬 @invoke in Chat, CLI, or GitHub.com
Define once → invoke anywhere
.github/agents/security-reviewer.agent.md
---
name: security-reviewer
description: Security-focused code reviewer
tools: ['grep_search', 'read_file', 'semantic_search']
handoffs:
  - label: Fix Issues
    agent: implementation
    prompt: Fix the security issues found above.
    send: false
model: gpt-5.4
---

You are a security expert. Review code for
OWASP Top 10 vulnerabilities. Flag SQL
injection, XSS, and auth bypasses.
  • Invoke with @security-reviewer in Chat
  • Define tools the agent can use
  • Set model preference per agent
  • Works in both VS Code & Copilot CLI

🏢 Org-level — .github-private repo for enterprise-wide rules
Layered: org → repo → file-scoped (all combine, no overrides)

MCP Servers

Connect external tools & data via the Model Context Protocol.

🗄️ External APIs · DBs · Services
↕
🔌 MCP Server (stdio / remote)
↕
🧠 Copilot Agent (any client)
Same server works in VS Code, CLI, JetBrains, etc.
.vscode/mcp.json
{
  "servers": {
    "github": {
      "type": "stdio",
      "command": "npx",
      "args": [
        "@modelcontextprotocol/server-github"
      ]
    }
  }
}
  • Tools auto-invoked in agent mode
  • Runs locally or as a remote service
  • No access to VS Code extension APIs
  • Works across different MCP clients

Hooks PREVIEW

Execute shell commands at key agent lifecycle points via JSON config files.

✏️ Agent edits a file
↓
🪝 PostToolUse hook fires
↓
✨ prettier · lint · security scan
Automate quality gates at every lifecycle event
.github/hooks/format.json
{
  "hooks": {
    "PostToolUse": [
      {
        "type": "command",
        "command": "npx prettier --write \"$TOOL_INPUT_FILE_PATH\""
      }
    ],
    "PreToolUse": [
      {
        "type": "command",
        "command": "./scripts/validate-tool.sh",
        "timeout": 15
      }
    ]
  }
}
EventFires WhenUse Case
SessionStartNew agent session beginsInject project context, log session start
UserPromptSubmitUser submits a promptAudit requests, inject system context
PreToolUseBefore agent invokes any toolBlock dangerous ops, require approval
PostToolUseAfter tool completes successfullyRun formatters, log results
PreCompactBefore context is compactedExport important context, save state
SubagentStartSubagent is spawnedTrack nested agents, init resources
SubagentStopSubagent completesAggregate results, cleanup
StopAgent session endsGenerate reports, send notifications
  • Generate with /create-hook or /hooks in Chat
  • Store in .github/hooks/*.json
  • Can also be scoped to a custom agent via hooks in frontmatter
  • Works across local agents, CLI, and cloud agents

2 · 💻 Copilot CLI

🎯

CLI Operating Modes

💬 Interactive Mode

Conversational session — ask questions, steer direction, give feedback. Copilot remembers context.

copilot

⚙️ Programmatic Mode

Single prompt in, result out — ideal for scripts & CI pipelines. Exits after completion.

copilot -p "fix the tests"

📐 Plan Mode

Copilot analyzes your request, asks clarifying questions, and builds a structured plan before writing code.

Shift+Tab to toggle

🚀 Autopilot Mode

Full autonomy — Copilot executes tools, edits files, and runs commands without per-step approval.

copilot --yolo
🖥️

Copilot CLI Overview

A powerful AI agent in your terminal — interactive chat, plan mode, and autonomous coding. Default model: Claude Sonnet 4.5.

copilot Launch interactive session (ask/execute mode)
copilot -p "task" Programmatic — run a single prompt, then exit
/model Switch AI model mid-session
Shift+Tab Toggle between ask/execute and plan mode
@path/to/file Include a file as context in your prompt
💡 Tip: Copilot CLI reads copilot-instructions.md, *.instructions.md, and AGENTS.md automatically.
🚀

Autopilot & Permissions

Control how much autonomy Copilot has with granular tool approvals.

🛡️ Ask each time
🔑 --allow-tool
🚀 --yolo
🔒 Most restricted 🔓 Full autonomy
Permission controls
# Full autonomy — all tools allowed
copilot --allow-all-tools
copilot --yolo  # alias

# Allow specific tools only
copilot --allow-tool='shell(git)'
copilot --allow-tool='write'

# Deny dangerous commands
copilot --allow-all-tools \
  --deny-tool='shell(rm)' \
  --deny-tool='shell(git push)'
⚠️ Caution: With --allow-all-tools, Copilot has the same access you do — review in sandboxed environments.

☁️ Runs in GitHub Actions (cloud)

👥 Team-visible — full audit trail
⏳ Async — works in background
🔒 Scoped to one repo per task
VS

🖥️ Agent Mode

💻 Runs in your local IDE
✏️ Edits files directly in editor
💬 Triggered from Chat in VS Code
👤 Personal — local changes only
⚡ Sync — real-time interaction
🔓 Access to entire workspace

Example: copilot-instructions.md for coding agent
# Project conventions for Copilot

## Build & Test
- Use pnpm (not npm or yarn)
- Run `pnpm test` before opening any PR
- Run `pnpm lint` to validate code style

## Code Style
- TypeScript strict mode — no any types
- React: functional components + hooks only
- Max file length: 300 lines

## PR Guidelines
- Include a summary of changes in PR body
- Reference the issue number in commits
💡 Tip: The coding agent runs in a GitHub Actions sandbox — it can install dependencies, run builds, and execute tests just like CI.

Tips & Best Practices

  • Start incremental — begin with /init, then add targeted instructions
  • Well-scoped prompts — include clear descriptions, acceptance criteria, and file hints
  • Specialize agents — create purpose-built agents for frontend, docs, testing, security
  • Review AI output — always review code from coding agent before merging
  • Use Copilot Memory — enable persistent repo knowledge for better context
  • Batch with skills — package multi-step workflows for reuse
  • Automate reviews — set up automatic Copilot code review via repo rules
  • Custom instructions for CR — add review-specific rules to copilot-instructions.md
