prompt engineering

The Goal-Context-Expectations Framework

🎯 Goal

What you want to accomplish. Be specific and focused.

"Write a function that validates email addresses"

📋 Context

Background, constraints, relevant files and patterns.

"Using validators.ts pattern, for React form"

✅ Expectations

Output format, quality criteria, edge cases.

"Include edge cases, use regex, add JSDoc"
💡 Pro Tip: Use chat variables like #file, #selection, @workspace, @terminal to provide rich context automatically.

review

Copilot Code Review

Open PR
→
Select Copilot
→
~30s Analysis
→
Inline Comments

✅ What it does

  • Leaves inline comments with suggested changes
  • Suggestions applied with a single click
  • Batch-commit multiple suggestions
  • Invokes cloud agent for complex fixes

⚠️ Key behaviors

  • Only leaves "Comment" reviews — never Approve
  • Does not count toward required approvals
  • Does not block merging
  • Re-review requires manual re-request

protocol

Model Context Protocol (MCP)

🔌

Open Standard

Defines how apps share context with LLMs. Standardized tool/resource connections.

🌐

Universal Access

Works in VS Code, JetBrains, CLI, Cloud Agent, Cursor, Windsurf, Xcode.

📦

MCP Registry

Curated community servers at github.com/mcp — browse and install with one click.

.vscode/mcp.json
// .vscode/mcp.json
{
  "mcpServers": {
    "github": { "type": "http", "url": "https://api.githubcopilot.com/mcp/" },
    "my-db": { "type": "stdio", "command": "npx", "args": ["@my-org/db-mcp"] }
  }
}
