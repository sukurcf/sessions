The Four Chat Modes

Choose the right mode for every task — from quick questions to full-project automation.

💬 Ask

Read-only Q&A. Get answers, explanations, and code suggestions without changing files.

✏️ Edit

Targeted inline edits. Apply changes to specific files you select, streamed as diffs.

🤖 Agent

Autonomous coding. Plans, edits files, runs terminal commands, and self-corrects.

📝 Plan

Think-first mode. Creates a step-by-step plan before writing any code.

📌 Shortcut: Switch modes with the dropdown at the top of the Chat panel, or type /ask, /edit, etc.

💬 Ask Mode & ✏️ Edit Mode

💬 Ask Mode

  • 📖 Read-only — never modifies files
  • 🔍 Explain code, debug errors, learn APIs
  • 📎 Attach files, selections, or images as context
  • 🌐 Use #web to search the internet
  • 🧪 Great for exploring unfamiliar codebases
💡 Tip: Use @workspace to ask questions about your entire project.

✏️ Edit Mode

  • 📝 Directly modifies files you specify
  • 🎯 You control which files are in scope
  • 📊 Streams changes as inline diffs
  • ↩️ Easy accept / reject per change
  • 🔒 No terminal access — safe, scoped edits
💡 Tip: Add files via Add Context button or drag them into chat.

Plan Mode — Think Before You Code

Generates a detailed step-by-step plan, then lets you review and refine before any code is written.

1 Analyze

Copilot reads your codebase and understands the full scope of the task

2 Plan

Generates a Markdown checklist with specific files and changes

3 Execute

After your approval, implements the plan step by step

🤖 Agent Mode

  • Starts coding immediately
  • Iterates and self-corrects as it goes
  • Best for well-defined tasks

📝 Plan Mode

  • Plans first, codes after approval
  • You can refine the plan before execution
  • Best for complex, multi-step refactors

Custom Instructions

Tell Copilot how to behave — coding standards, preferred patterns, and project conventions.

📁 Repo-Level Instructions

.github/copilot-instructions.md

Shared across the team via version control. Define coding style, architecture patterns, testing conventions.

👤 User-Level Instructions

VS Code Settings → github.copilot.chat.codeGeneration.instructions

Personal preferences that apply across all repositories — your preferred language, style, etc.

📄 Instruction Files NEW

Use .instructions.md files anywhere in your workspace with front-matter applyTo globs to scope instructions to specific file types or directories.

💡 Pro Tip: Combine all three levels — repo-level for team standards, user-level for personal style, and instruction files for targeted rules.

Custom Agents & Tools

🤖 Custom Agents

  • 🏗️ Defined via .github/agents/*.md
  • 🎯 Scoped instructions for specific domains
  • 🔧 Can attach specific tools & MCP servers
  • 📎 Include custom system prompts
  • ⚡ Invoke with @agent-name in chat

🔧 Tools & MCP

  • 🌐 MCP (Model Context Protocol) servers
  • 🔌 VS Code extensions as tool providers
  • 📦 Built-in: terminal, file ops, search
  • 🗄️ Connect DBs, APIs, cloud services
  • ⚙️ Configure in settings.json or .vscode/mcp.json
