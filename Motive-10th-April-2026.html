1 · 📋 Custom Instructions

📋

Instruction Types & Scopes GA

Teach Copilot your project conventions with always-on, file-scoped, and organization-level rules.

🌐 Always-on — copilot-instructions.md · AGENTS.md · CLAUDE.md
🎯 File-scoped — *.instructions.md with applyTo glob patterns
🏢 Org-level — .github-private repo for enterprise-wide rules
Layered: org → repo → file-scoped (all combine, no overrides)
copilot-instructions.md Always-on — applies to every request in the repo
*.instructions.md File-based — scoped by applyTo glob pattern
AGENTS.md Always-on — cross-client compatible rules
CLAUDE.md Always-on — Claude-specific rules
🎯

File-Based Instructions

Scope rules to specific files using glob patterns in the YAML frontmatter.

.github/instructions/react.instructions.md
---
applyTo: "src/**/*.tsx"
---

Use React functional components with hooks.
Prefer useReducer over useState for
complex state. Always add aria-* attributes
for accessibility.
.github/instructions/python.instructions.md
---
applyTo: "**/*.py"
---

Use 4-space indentation, type hints on all
public functions, and pathlib for file paths.
  • Create with /create-instruction or /init
  • Multiple instruction files can apply to the same file
  • All matching instructions combine — no overrides
🏢

Organization Custom Instructions GA

Set enterprise-wide coding standards that apply across all repositories in your organization.

🏢 .github-private repo
↓
📋 copilot-instructions.md
↓
✅ Applied to all org repos
Organization → Repository → File-scoped (all layers combine)
  • Create a .github-private repo in your org
  • Add copilot-instructions.md to the root
  • Rules apply to all repos in the organization
  • Repository-level instructions layer on top
📝

Example: copilot-instructions.md

.github/copilot-instructions.md
# Project conventions for Copilot

## Build & Test
- Use pnpm (not npm or yarn)
- Run `pnpm test` before opening any PR
- Run `pnpm lint` to validate code style

## Code Style
- TypeScript strict mode — no any types
- React: functional components + hooks only
- Max file length: 300 lines

## PR Guidelines
- Include a summary of changes in PR body
- Reference the issue number in commits
💡 Tip: Start with /init to auto-generate a copilot-instructions.md tailored to your codebase.

2 · 📐 Planning Mode

📐

Plan Agent GA

Copilot analyzes your request, asks clarifying questions, and builds a structured implementation plan before writing any code.

📝 Your prompt
↓
🧠 Copilot analyzes & plans
↓
📋 Structured plan generated
↓
✅ You approve → execution begins
Think first, code second — review the approach before any changes
  • Select Plan from the agent picker in VS Code Chat
  • In CLI: press Shift+Tab to toggle plan mode
  • Plan outlines files to create/modify, approach, and dependencies
  • Review and approve before Copilot starts coding
🎯

When to Use Planning Mode

ScenarioWhy Plan Mode Helps
🏗️ Multi-file featuresSee which files will be created/modified before any changes
🔄 RefactoringReview the refactoring strategy — ensure nothing breaks
🧪 Test generationConfirm testing approach, frameworks, and coverage targets
📦 New modulesApprove the architecture and file structure upfront
🔀 Complex migrationsValidate migration steps before touching production code
📌 Key: Planning mode is ideal when you want Copilot to show its reasoning and approach before making changes.
☁️

Cloud Agent Planning NEW

Ask the Copilot cloud agent to generate an implementation plan and review the approach before any code is written.

💬 "Create a plan for…"
↓
🤖 Copilot generates plan
↓
👁️ You review & approve
↓
⚙️ Copilot implements plan
Plan → Review → Approve → Implement
  • Available via cloud agent on GitHub.com
  • Ask for a plan in your prompt
  • Review Copilot's proposed approach before any code
  • Once approved, Copilot uses the plan to guide implementation
💻

Plan Mode in Copilot CLI

Toggle between ask/execute and plan mode in the terminal.

Shift+Tab Toggle between ask/execute and plan mode
copilot -p "plan: ..." Programmatic mode with plan request
/plan Switch to plan mode mid-session
💡 Tip: Use plan mode for complex tasks and switch to execute mode for quick fixes.

3 · 🤖 Agent Mode

🤖

Agent Mode Overview GA

Agent mode gives Copilot autonomy to plan, edit files, run terminal commands, and iterate — all within your local VS Code workspace.

💬 Your prompt
↓
🧠 Copilot plans approach
↓
✏️ Edits files in workspace
↓
⚡ Runs terminal commands
↓
🔄 Iterates until complete
Autonomous coding — Copilot drives, you review
  • Select Agent from the agent picker in Chat
  • Full access to workspace files and integrated terminal
  • Uses tools: file read/write, search, terminal, browser
  • Creates checkpoints for easy rollback
🎯

Session Types

Choose where the agent runs based on your workflow needs.

Local Runs interactively in your editor with full workspace access
Background Runs in background — continue working while Copilot codes
Cloud Runs in GitHub Actions environment — fully remote
📌 Note: Background and Cloud sessions free up your editor while Copilot works asynchronously.
🔐

Permission Levels

Control how much autonomy the agent has over tool approvals.

🛡️ Ask
🔑 Allow
🚀 Yolo
🔒 Most restricted 🔓 Full autonomy
LevelBehavior
Ask each timeCopilot requests approval for every tool use
Allow for sessionAuto-approve specific tool categories
Yolo modeFull autonomy — all tools allowed without approval
⚠️ Caution: With full autonomy, Copilot has the same access you do — review in sandboxed environments.

6 · 👁️ Code Review Agent

👁️

Copilot Code Review GA

AI-powered code review with inline suggestions and one-click fixes on every pull request.

🔀 Open PR
→
🤖 Copilot Reviews
→
💬 Inline Comments
→
✅ One-Click Fix
Automatic or manual — surfaces issues, never blocks merging
Request review Select "Copilot" from the Reviewers menu on any PR
Automatic reviews Configure via repository rulesets for all PRs
Suggested changes Apply fixes with one click or batch commit
Implement suggestion Invoke cloud agent to apply complex multi-file fixes
📌 Note: Copilot leaves "Comment" reviews — it never "Approves" or "Requests changes" and won't block merging.
🔍

TypeHow It Works

⚡ Automatic reviewConfigure via repo rulesets — Copilot reviews every PR automatically
🎯 Targeted reviewSelect specific lines/files for Copilot to focus on
💡 Tip: Combine automatic Copilot reviews with human reviewers for comprehensive coverage.

Customizing Code Review

Shape what Copilot looks for during code review using custom instructions.

Review-specific instructions in copilot-instructions.md
## Code Review Guidelines

- Flag any use of any type in TypeScript
- Ensure all API endpoints have input validation
- Check for missing error handling in async code
- Verify test coverage for new functions
- Flag hardcoded secrets or credentials
  • Add review-specific rules to copilot-instructions.md
  • Copilot follows your coding standards during reviews
  • Works with both manual and automatic reviews
🔧
