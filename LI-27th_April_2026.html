Sub-agents Key Concept

Independent agents that work in isolated context windows — research, analyze, and return only a focused summary.

🧊 Context Isolation

Each sub-agent gets its own context window. Does NOT inherit the parent's conversation history — receives only the task prompt.

⚡ Parallel Execution

VS Code spawns multiple sub-agents simultaneously — e.g., security + performance + accessibility analysis in one pass.

📊 Focused Results

Only the final summary returns to the parent agent. Intermediate steps stay in the sub-agent's context — keeps parent lean.

🔄 Synchronous Wait

Main agent waits for sub-agent results before continuing. Findings directly inform the next decision or step.

Sub-agent Delegation Flow

Parent agent decomposes tasks → delegates to specialized sub-agents → aggregates results

👤 User Prompt
"Refactor auth module"
↓
🧠 Main Agent (Plan)
Decomposes into sub-tasks
↓   ↓   ↓
🔎 Search Agent
Find auth files & deps
🛡️ Security Agent
Audit vulnerabilities
🏃 Test Agent
Run existing tests
↓ summaries only ↓
✅ Main Agent Aggregates & Implements
Uses summaries to write code → run tests → verify
💡 Why this works: Sub-agent research doesn't pollute the main context window. Only concise summaries come back — keeping the main agent focused and efficient.

Context Window — 7 Layers

Everything the model can see when generating a response. What's outside is invisible.

1
System Instructions — Agent behavior rules
2
Customizations — Skills, custom instructions, AGENTS.md
3
User Message — Current prompt
4
Conversation History — Previous turns
5
Implicit Context — Active file, selection, errors
6
Explicit References — #file, #codebase, #fetch
7
Tool Outputs — Terminal, search, file reads

Context Optimization

Strategies to maximize signal-to-noise in your context window

🔀
Sub-agents
Isolate research in separate context — return summary only
📦
Progressive Skill Loading
3-level: name → instructions → resources (on demand)
🗜️
Auto Compaction
VS Code summarizes older messages transparently
📌
Explicit #-mentions
Reference specific files instead of entire codebase
🔧
Tool Sets
Group tools, enable only relevant ones per request
🆕
New Chat Session
Fresh context for unrelated tasks — maximum savings

❌ Don't

  • Dump entire codebase into context
  • Keep stale conversation history
  • Load all tools for every request

✅ Do

  • Use #file for targeted context
  • Run /compact with focus instructions
  • Delegate research to sub-agents
