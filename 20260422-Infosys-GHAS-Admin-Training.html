GitHub Advanced Security — Overview

GHAS is now split into two purchasable products with separate licensing

🔑 Secret Protection

  • Secret scanning alerts
  • Push protection
  • Copilot secret scanning (AI-powered)
  • Custom secret patterns
  • Delegated bypass controls
  • Security campaigns (secrets)

🛡️ Code Security

  • Code scanning (CodeQL)
  • Copilot Autofix
  • CodeQL CLI
  • Custom Dependabot auto-triage rules
  • Dependency review
  • Security campaigns (code)
💡 Pro Tip: Many features are free on public repositories — code scanning, secret scanning, push protection, and Copilot Autofix.

AI-Powered Security with Copilot

Copilot Autofix

LLM-powered code fix suggestions generated directly on code scanning alerts. Supports C#, Go, Java, Python, JS/TS, Ruby, Rust, Swift, C/C++.

Copilot Secret Scanning

AI detection of unstructured secrets — passwords, connection strings, and generic API keys that standard regex patterns miss.

Copilot Cloud Agent

In security campaigns, Copilot automatically creates PRs for alerts with autofixes and assigns the campaign manager as reviewer.

Audit Logs & Monitoring

Audit Logs

Complete record of all administrative and security-relevant actions across the organization. Filterable, searchable, and exportable.

Log Streaming

Stream audit data to SIEM tools — Splunk, Datadog, Azure Sentinel, AWS S3, Google Cloud Storage for real-time correlation.

Webhooks

Automated triggers for security events — secret detected, alert opened, push protection bypassed. Powers custom integrations.

Automation

Build monitoring workflows with GitHub Actions — auto-notify on bypasses, enforce SLAs, and escalate unresolved alerts.

📌 Key: Push protection bypass events are logged with the bypass reason, enabling compliance auditing and pattern analysis.

Secret Scanning & Push Protection

Secret Scanning

  • Scans entire Git history on all branches
  • Issues, PRs, discussions, wikis, gists
  • Partner program → auto-revocation
  • Custom regex patterns for org-specific secrets
  • Validity checks verify if credentials are active

Push Protection

  • Blocks pushes containing secrets before they land
  • CLI, UI, API, REST, and MCP interactions
  • User-level (all GitHub.com users, on by default)
  • Repo-level (requires Secret Protection license)
  • Delegated bypass with approval workflows

Security Campaigns

Group security alerts and collaborate with developers for remediation at scale

Create

Use templates to select groups of related alerts. Available via Security Overview UI or REST API.

Assign

Assign alerts to developers (write access) or Copilot Cloud Agent for automatic PR creation.

Track

Monitor progress through the Campaigns view. Named point-of-contact for questions and reviews.

Code Scanning Campaigns

  • Default branch alerts only
  • Repository tracking issues
  • Copilot Autofix auto-triggered
  • Requires write access

Secret Scanning Campaigns PREVIEW

  • All alerts (not just default branch)
  • No tracking issues
  • No automatic remediation
  • May elevate permissions temporarily

Admin Best Practices Checklist

✅ Security Enablement

  • Enable push protection org-wide
  • Configure default security settings for new repos
  • Enable CodeQL default setup on all repos
  • Enable Copilot Autofix (on by default)

✅ Governance

  • Enforce 2FA for all org members
  • Configure delegated bypass for push protection
  • Set up audit log streaming to SIEM
  • Verify organization domains

✅ Operations

  • Create custom secret patterns for internal tokens
  • Review Security Overview dashboards weekly
  • Launch security campaigns for critical vulns
  • Define alert ownership and SLA models

✅ Monitoring

  • Set up webhook alerts for bypass events
  • Track enablement trends across teams
  • Monitor Copilot Autofix adoption metrics
  • Export coverage reports for compliance
